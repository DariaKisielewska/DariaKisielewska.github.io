Virtual environment

June 10, 2020
python virtualenv

Why?

Virtual environment keeps dependencies required in different projects in separate places. This solution make easier to maintain different projects.

How?

Installation (for Fedora otherwise use pip):

sudo yum install python-virtualenv

Creating new env:

virtualenv -p /path/to/a/python.exe  /path/to/env_name

Activating:

source /path/to/env_name/bin/activate

After finishing the project

deactivate

Where to store virtual environments?

  • with the project (add folder to the .gitignore)

  • in home directory. Then you can control them by virtualenvwrapper

Auto-activating

Use directory-based environments like autoenv or direnv.

Manage different python versions

pythonXX -m pip
