Documentation in Sphinx

June 16, 2020
python sphinx

Goal

Set-up Sphinx as a documentation engine for the project. Instead of visually dense reStructuredText let’s use Google docstring. Support for this function is provided by sphinx.ext.napoleon extension.

At the end, substitute the standard theme by sphinx-rtd-theme.

Install pre-requisites

python3.8 -m pip install sphinx sphinxcontrib-napoleon sphinx-rtd-theme

Set-up documentation

cd projectdir
mkdir docs
cd docs
sphinx-quickstart

sphinx-quickstart asks for basics information about the project. Keep sources and build directories separate.

Configuration file is stored in docs/source/conf.py. In next step configure autodoc, by uncommenting following lines:

import os
import sys
sys.path.insert(0, os.path.abspath('.'))

and changing last one to

sys.path.insert(0, os.path.abspath('../..'))

Change

extensions = []

to

extensions = ['sphinx.ext.autodoc', 'sphinx.ext.napoleon']

Change theme html_theme = ‘alabaster’ to html_theme = ‘sphinx_rtd_theme’

Use sphinx-apidoc to build the API documentation

sphinx-apidoc -f -o source ../projectdir

Finally, generate the HTML: make html. The files are located in docs/build/html.

Other thoughts

In Python 3.5+ Type Hinting will reduce needed commenting.
